# Бот «Тест на грейд графического дизайнера»

Телеграм-бот: пускает к тесту только подписчиков вашего канала, задаёт 15 ситуационных вопросов («как бы вы поступили»), за каждый ответ начисляет 0–3 балла и по сумме выдаёт грейд — от Junior− до Senior+ (9 уровней).

## Файлы

| Файл | Что это |
|---|---|
| `bot.py` | Логика бота |
| `questions.py` | Вопросы, баллы и грейды — редактируйте здесь |
| `requirements.txt` | Зависимости |

## Шаг 1. Создать бота

1. Откройте в Telegram **@BotFather** → команда `/newbot`.
2. Придумайте имя и юзернейм (заканчивается на `bot`).
3. Скопируйте **токен** — он понадобится дальше.

## Шаг 2. Дать боту доступ к каналу

Чтобы бот мог проверять подписку, добавьте его **администратором** вашего канала (никакие права можно не включать — достаточно самого статуса админа).

Канал должен быть **публичным** (с юзернеймом вида `@my_channel`).

## Шаг 3. Запустить

Бот настраивается двумя переменными окружения:

- `BOT_TOKEN` — токен от BotFather
- `CHANNEL` — юзернейм канала, например `@my_design_channel`

Локально (для проверки, нужен Python 3.10+):

```bash
pip install -r requirements.txt

# Windows (PowerShell):
$env:BOT_TOKEN="123456:ABC..."
$env:CHANNEL="@my_design_channel"
python bot.py

# macOS / Linux:
BOT_TOKEN="123456:ABC..." CHANNEL="@my_design_channel" python bot.py
```

Пока окно терминала открыто — бот работает. Для работы 24/7 нужен хостинг.

## Шаг 4. Хостинг 24/7

Полностью бесплатных вариантов без ограничений сейчас почти нет, реальные варианты:

- **Bot-Hosting / Bothost** — есть бесплатный план для небольших ботов (128 MB RAM хватит), деплой из GitHub.
- **PythonAnywhere** — бесплатный тариф не держит ботов постоянно; тариф ~$5/мес — держит (Always-on task).
- **Railway / Render** — бесплатные планы с ограничениями: сервис может «засыпать» без трафика.
- **Дешёвый VPS** (~150–300 ₽/мес) — самый надёжный вариант: `pip install -r requirements.txt`, запуск через `systemd` или `screen`.

Везде схема одна: загрузить три файла, указать переменные `BOT_TOKEN` и `CHANNEL`, команда запуска — `python bot.py`.

## Как править тест

Всё в `questions.py`:

- **Вопросы** — список `QUESTIONS`. У каждого варианта ответа свой балл (0–3). Можно добавлять и удалять вопросы — грейд считается от процента, пересчитывать ничего не надо.
- **Грейды** — список `GRADES`: процентные пороги, названия и описания.
- Варианты ответов автоматически перемешиваются у каждого пользователя, чтобы нельзя было поделиться «правильной буквой».

## Особенности

- Подписка проверяется дважды: на входе и перед выдачей результата (отписаться посреди теста не получится).
- Прогресс хранится в памяти: при перезапуске бота незаконченные тесты сбрасываются (пользователь просто жмёт /start заново). Для простого теста этого достаточно.
